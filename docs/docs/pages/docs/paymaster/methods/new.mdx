# New

Creates a new Paymaster client instance for interacting with a SNIP-29 compliant paymaster service. The paymaster service sponsors transaction fees, allowing users to pay with alternative tokens or have their fees covered entirely. The client automatically maintains cookies for session management.

## Function Signature

```go
func New(ctx context.Context, url string, options ...client.ClientOption) (*Paymaster, error)
```

**Source:** [paymaster.go:L64-L80](https://github.com/NethermindEth/starknet.go/blob/main/paymaster/paymaster.go#L64-L80)

## Parameters

- `ctx` (context.Context): Context for the operation
- `url` (string): URL of the paymaster service
- `options` (...client.ClientOption): Optional client configuration options

## Returns

- `*Paymaster`: New paymaster client instance
- `error`: Error if client creation fails

## Usage Example

```go
package main

import (
	"context"
	"fmt"
	"log"

	"github.com/NethermindEth/starknet.go/paymaster"
)

func main() {
	// Using AVNU's public SNIP-29 compliant paymaster on Sepolia testnet
	paymasterURL := "https://sepolia.paymaster.avnu.fi"
	ctx := context.Background()

	// Create a new paymaster client
	pm, err := paymaster.New(ctx, paymasterURL)
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println("Paymaster client created successfully")

	// Check if the paymaster service is available
	available, err := pm.IsAvailable(ctx)
	if err != nil {
		log.Fatal(err)
	}

	if available {
		fmt.Println("Paymaster service is ready")
	}
}
```

## Common Use Cases

- Create a paymaster client for gasless transactions where users don't need ETH/STRK for gas.
- Enable alternative fee token payments with USDC, DAI, or other supported ERC-20 tokens.
- Build sponsored dApps where the application covers user transaction fees.
- Requires a SNIP-29 compliant paymaster service with the URL pointing to the JSON-RPC endpoint.
- Client automatically maintains cookies for session management.
