# TrackingIDToLatestHash

Retrieves the latest transaction hash associated with a tracking ID. Maps a tracking ID (received from BuildTransaction) to the latest transaction hash, useful for monitoring transaction status and handling retries if a transaction fails.

## Method Signature

```go
func (pm *Paymaster) TrackingIDToLatestHash(
    ctx context.Context,
    trackingID *felt.Felt,
) (TrackingIDResponse, error)
```

**Source:** [tracking_id.go:L26-L41](https://github.com/NethermindEth/starknet.go/blob/main/paymaster/tracking_id.go#L26-L41)

## Parameters

- `ctx` (context.Context): Context for the operation
- `trackingID` (*felt.Felt): Tracking ID from BuildTransaction

## Returns

- `TrackingIDResponse`: Response containing the transaction hash and status
- `error`: Error if tracking ID is invalid or not found

## Usage Example

```go
// Get tracking ID from BuildTransaction
buildResp, _ := pm.BuildTransaction(ctx, request)
trackingID := buildResp.TrackingID

// Later, check transaction status
response, err := pm.TrackingIDToLatestHash(ctx, trackingID)
if err != nil {
    log.Fatal(err)
}

fmt.Printf("Transaction Hash: %s\n", response.TransactionHash)
fmt.Printf("Status: %s\n", response.Status)
```

## Common Use Cases

- Monitor transaction status by mapping tracking IDs to their latest transaction hashes.
- Implement retry management by checking transaction status and re-submitting if necessary.
- Provide real-time status updates to users during long-running transaction processing.
- TrackingID is obtained from BuildTransaction response and links all operations for that transaction.
- If a transaction is replaced or retried by the paymaster, this method returns the latest hash.
- Status values and behavior depend on the specific paymaster implementation.
