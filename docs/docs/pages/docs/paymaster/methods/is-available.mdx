# IsAvailable

Checks if the paymaster service is running and accepting requests. This performs a health check to determine if the service is operational and ready to process transactions. This should be called before attempting to build or execute transactions.

## Method Signature

```go
func (pm *Paymaster) IsAvailable(ctx context.Context) (bool, error)
```

**Source:** [is_available.go:L14-L21](https://github.com/NethermindEth/starknet.go/blob/main/paymaster/is_available.go#L14-L21)

## Parameters

- `ctx` (context.Context): Context for the operation

## Returns

- `bool`: True if service is available, false otherwise
- `error`: Error if the check fails

## Usage Example

```go
package main

import (
	"context"
	"fmt"
	"log"

	"github.com/NethermindEth/starknet.go/paymaster"
)

func main() {
	// Connect to AVNU's public paymaster service on Sepolia testnet
	paymasterURL := "https://sepolia.paymaster.avnu.fi"
	ctx := context.Background()

	fmt.Printf("Paymaster Health Check Example\n")

	// Create paymaster client
	pm, err := paymaster.New(ctx, paymasterURL)
	if err != nil {
		log.Fatalf("Failed to create paymaster client: %v", err)
	}

	fmt.Println("Paymaster client created")
	fmt.Println("\nPerforming health check...")

	// Check if paymaster service is available and operational
	available, err := pm.IsAvailable(ctx)
	if err != nil {
		log.Fatalf("Health check failed: %v", err)
	}

	// Display results
	fmt.Println()
	if available {
		fmt.Println("Paymaster service is AVAILABLE")
		fmt.Println("\nService Status: OPERATIONAL")
		fmt.Println("Ready to process transactions")
	} else {
		fmt.Println("Paymaster service is UNAVAILABLE")
		fmt.Println("\nService Status: DOWN")
		fmt.Println("The service is temporarily unavailable")
	}
}
```

## Common Use Cases

- Perform startup health checks before attempting to build or execute transactions.
- Implement retry logic with exponential backoff when the service is temporarily unavailable.
- Monitor service availability for operational metrics and alerting systems.
- Returns false (not error) if service is temporarily unavailable, while errors indicate network or communication issues.
- Lightweight operation suitable for frequent health checks.
