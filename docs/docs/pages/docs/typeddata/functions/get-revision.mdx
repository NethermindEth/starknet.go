# GetRevision

Returns the revision object for a given version number (0 or 1).

## Function Signature

```go
func GetRevision(version uint8) (rev *revision, err error)
```

**Source:** [revision.go:L124-L133](https://github.com/NethermindEth/starknet.go/blob/main/typeddata/revision.go#L124-L133)

## Parameters

- `version` (uint8): Revision version (0 for Pedersen, 1 for Poseidon)

## Returns

- `*revision`: Pointer to the revision object containing hash functions and type definitions
- `error`: Error if version is invalid (not 0 or 1)

## Usage Example

```go
package main

import (
	"fmt"
	"log"

	"github.com/NethermindEth/starknet.go/typeddata"
)

func main() {
	// Get Revision 1 (Poseidon hashing)
	_, err := typeddata.GetRevision(1)
	if err != nil {
		log.Fatalf("Invalid revision: %v", err)
	}

	fmt.Printf("Revision 1 loaded\n")
	fmt.Printf("Uses Poseidon hashing\n")
	fmt.Printf("Supports: u128, i128, ContractAddress, enums, u256 preset\n")

	// Get Revision 0 (Pedersen hashing)
	_, err = typeddata.GetRevision(0)
	if err != nil {
		log.Fatalf("Invalid revision: %v", err)
	}

	fmt.Printf("\nRevision 0 loaded\n")
	fmt.Printf("Uses Pedersen hashing\n")
	fmt.Printf("Supports: felt, bool, string, selector\n")
}
```

## Common Use Cases

- Determine which revision a TypedData instance uses for version detection. See [Working with Revision 1 example](/docs/typeddata/examples#working-with-revision-1).
- Validate that your application supports the required revision for compatibility checks.
- Handle different revisions in applications supporting both versions during migration workflows.
- Revision 1 is recommended for new applications due to Poseidon's efficiency and expanded type support.
