# DataAvailabilityModeConc

import { Callout } from 'vocs/components'

<Callout type="warning">
  **Internal Helper Function**: This is a low-level utility used internally by V3 transaction hash functions. Most developers should use [TransactionHashInvokeV3](./transaction-hash-invoke-v3), [TransactionHashDeclareV3](./transaction-hash-declare-v3), or [TransactionHashDeployAccountV3](./transaction-hash-deploy-account-v3) instead.
</Callout>

Concatenates data availability modes for fee and nonce into a single uint64 value. This internal helper function combines the fee data availability mode and nonce data availability mode for V3 transaction hash calculations.

## Function Signature

```go
func DataAvailabilityModeConc(
    feeDAMode rpc.DataAvailabilityMode,
    nonceDAMode rpc.DataAvailabilityMode,
) (uint64, error)
```

**Source:** [hash.go:L788-L800](https://github.com/NethermindEth/starknet.go/blob/main/hash/hash.go#L788-L800)

## Parameters

- `feeDAMode` (rpc.DataAvailabilityMode): Fee data availability mode (L1 or L2)
- `nonceDAMode` (rpc.DataAvailabilityMode): Nonce data availability mode (L1 or L2)

## Returns

- `uint64`: The concatenated data availability modes value
- `error`: Error if conversion fails

## Usage Example

```go
package main

import (
    "fmt"
    "log"

    "github.com/NethermindEth/starknet.go/hash"

func main() {
    // This is an internal helper - typically you would use specific V3 transaction hash functions

    feeDAMode := rpc.DataAvailabilityMode(0)    // L1 data availability
    nonceDAMode := rpc.DataAvailabilityMode(0)  // L1 data availability

    if err != nil {
```

## Common Use Cases

This internal function is commonly used for:

- Computing the data availability mode component when calculating V3 invoke transaction hashes to encode fee and nonce DA settings.
- Generating the DA mode value for V3 declare transactions to specify whether transaction data is posted to L1 or L2 for verification.
- Calculating the DA mode component for V3 deploy account transactions to control gas costs through data availability choices.
- Implementing transaction analysis tools that decode and validate data availability settings from V3 transaction hashes.
- Building fee estimation systems that need to understand how data availability mode choices affect transaction costs and security guarantees.
