# MerkleHash

Calculates the Merkle hash of two big integers using the Pedersen hash function. This function ensures deterministic ordering by always hashing the smaller value first, which means `MerkleHash(a, b)` always equals `MerkleHash(b, a)`.

## Function Signature

```go
func MerkleHash(x, y *big.Int) *big.Int
```

**Source:** [merkle.go:L46-L52](https://github.com/NethermindEth/starknet.go/blob/main/merkle/merkle.go#L46-L52)

## Parameters

- `x` (*big.Int): The first big integer value
- `y` (*big.Int): The second big integer value

## Returns

- `*big.Int`: The Merkle hash result as a big integer

## How It Works

The `MerkleHash` function implements a deterministic hashing scheme:

1. Compares the two input values `x` and `y`
2. Orders them so the smaller value is always first
3. Computes the Pedersen hash using `curve.HashPedersenElements`
4. Returns the resulting hash

This ordering ensures that `MerkleHash(a, b)` always equals `MerkleHash(b, a)`, which is important for Merkle tree construction and proof verification.

## Usage Example

```go
package main

import (
	"fmt"
	"math/big"

	"github.com/NethermindEth/starknet.go/merkle"
)

func main() {
	// Create two big integers to hash
	x := big.NewInt(100)
	y := big.NewInt(200)

	// Calculate Merkle hash
	hash := merkle.MerkleHash(x, y)

	fmt.Println("MerkleHash:")
	fmt.Printf("  Input X: %s\n", x.String())
	fmt.Printf("  Input Y: %s\n", y.String())
	fmt.Printf("  Merkle Hash: %s\n", hash.String())
}
```

## Common Use Cases

- `MerkleHash` is the fundamental building block for constructing Merkle trees. See [Manual Tree Construction example](/docs/merkle/examples#manual-merkle-tree-construction).
- When verifying a Merkle proof, you use `MerkleHash` to recompute intermediate hashes along the path from the leaf to the root.
- In Starknet, `MerkleHash` is used to create compact commitments to larger data structures, allowing efficient verification of state.
- The function ensures consistent hash values regardless of input order, making it reliable for cryptographic operations. See [Deterministic Hash Ordering example](/docs/merkle/examples#deterministic-hash-ordering).
